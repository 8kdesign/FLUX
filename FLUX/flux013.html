FLUX #013 - TPG

16 December 2020

TPG is the 4th telco in Singapore and they just released their app not long ago. As a subscriber to their services, I naturally downloaded the app, but found it really frustrating to use. Checking the Google Play Store, I found out that I wasn't the only one. Hence, in this FLUX, let's see how we can address that.

Issue #1: Troublesome Login Process

Every time you log in to the app, you have to key your TPG phone number, followed by an SMS authentication code. I get the authentication code, but why can't the app remember my phone number? As a secondary SIM just for data, I can barely remember my TPG number. Also, that's a lot of typing just to log in.

To make things worse, if you accidentally press back after login in (on the main page), you will be logged out and have to log in all over again. Looks like someone did not implement this part properly.

Issue #2: Duplicate Menu

There are 2 ways to navigate around the app: a bottom navigation bar and a hamburger menu. And the options you get are basically the same, with the hamburger menu having slightly more. What is the point of having 2 different menus if they more or less bring you to the same pages?

Having a hamburger menu on the very top left corner is really hard to reach too. The same goes for the notification toggle on the top right corner. Hence for my concept, I will stick to the bottom navigation bar.

Notice the option "Usage" in the hamburger menu? That can be accessed via the dashboard, but only if you click on the icons (and not the words). If I wasn't playing around with the app, I would not know it was even there. 

Issue #3: Wasted Pages

There are a couple of pages that can be excluded to provide a better experience. For example, the support page only has links to 2 external sites. The profile page is really bare too, and all of these could actually come together to form a settings page. The legal and logout toggles could also be added to the settings page.

Redesigned Login System

The biggest frustration most users have with the app is logging in and that can actually be fixed easily, without compromising security. On the first login, the user can be greeted with the main login screen with phone number, but subsequent logins will only ask for the OTP. I also added an option to switch account here.

New Nav Menu & Dashboard

Simplifying the menu, we now only have one menu with 4 options: Dashboard, Billing, Notification and Settings.

Topping up your account is done directly on the dashboard, so the billing page now only contain a list of receipt. This would save the user an additional tap. I added progress bars to the stats for easier visualization and also a "Analyze Usage" toggle that would bring users to the more detailed usage page.

Usage Page

To make the app feel more lite, I made the usage page (and all additional layers) a popup. I also shifted down the local/international toggle for greater reach-ability. The same was not done for the tabs as the user can simply swipe left/right on any part of the popup to switch between "data" and "talk & text".

For the data usage graph, I changed its orientation and size so scrolling is no longer required. Additionally, the new graph would only show usage for the current month, and not days from the previous months. 

Billing & Notification Page

As mentioned earlier, the billing page has been simplified to only show the receipts. Tapping on the listing would bring up a popup showing the receipt. Previously, you would get 2 receipts every time, one under billing and another under notification, both with different information. The new receipt combines both.

And in the new notification page, I removed all notifications for emailed receipts. Notice how I removed the timing for each listing? It really didn't make sense to me to simplify the date to number of days passed, yet keep a detail so specific like the time posted. We have a similar popup message when tapped.

Settings Page

Everything else can be accessed via the settings page. These are miscellaneous stuff, things that users would not interact with often (hopefully). Since there are not a lot of such toggles, having them all on the same page would help the user to find them easier and at the same time make the app a lot simpler.
